# کتاب‌باز — راهنمای معماری جدید (نسخه چند صفحه‌ای + سئو)

## این چیه و چی فرق کرد؟

از الان سایتت یک **اسکریپت ساخت خودکار** (`build.js`) دارد که هر بار push می‌کنی، خودش:
- برای **هر کتاب** یک صفحه HTML واقعی و مجزا می‌سازد → `books/{slug}.html`
- برای **هر کتاب صوتی** یک صفحه پخش مجزا می‌سازد → `audiobooks/{slug}.html`
- صفحه لیست کتاب‌ها، صفحه پادکست (با پخش‌کننده mp3 واقعی)، و صفحه خانه (فقط جدیدترین‌ها) را می‌سازد
- `sitemap.xml` را خودش از نو می‌سازد — هیچوقت دستی لازم نیست دستکاریش کنی
- کتاب‌های مرتبط (همون ژانر) را در پایین هر صفحه کتاب لینک می‌دهد

**کار روزانه‌ات تغییر نمی‌کند** — هنوز فقط فایل‌های JSON توی پوشه `data/` را ویرایش می‌کنی. تنها فرقش این است که حالا هر بار push می‌کنی، Cloudflare Pages خودش این اسکریپت را اجرا می‌کند و ده‌ها صفحه واقعی می‌سازد.

## ساختار پوشه‌ها

```
ketabbaaz-site/
├── build.js              ← اسکریپت ساخت‌دهنده (دست نزن)
├── package.json          ← فقط برای این‌که Cloudflare بداند چطور build کند
├── data/
│   ├── books.json         ← هر روز کتاب جدید اینجا اضافه می‌کنی
│   ├── facts.json
│   ├── excerpts.json
│   ├── podcast.json       ← قسمت‌های پادکست
│   └── audiobooks.json    ← کتاب‌های صوتی
├── static/
│   └── style.css          ← استایل کل سایت (دست نزن مگر بخوای ظاهر را عوض کنی)
├── files/                 ← فایل‌های PDF کتاب‌ها
└── audio/                 ← فایل‌های mp3 (پادکست و کتاب صوتی)
```

⚠️ **پوشه `dist/` رو خودت نمی‌سازی** — Cloudflare Pages هر بار خودش با اجرای `build.js` این پوشه را تازه می‌سازد و منتشر می‌کند.

## تنظیمات Cloudflare Pages (یک‌بار باید عوض کنی)

برو به پروژه‌ات در Cloudflare Pages → **Settings → Builds & deployments** و این دو مقدار را تنظیم کن:

| تنظیم | مقدار |
|---|---|
| Build command | `node build.js` |
| Build output directory | `dist` |

بعد از ذخیره، یک Deploy جدید بزن تا اولین ساخت کامل انجام شود.

## یک کار مهم دیگر: آدرس دامینت را توی build.js بگذار

فایل `build.js` را باز کن (در خود گیت‌هاب، با مداد ویرایش)، خط دوم رو پیدا کن:

```js
const SITE_URL = 'https://YOUR-DOMAIN.com';
```

`YOUR-DOMAIN.com` را با دامین واقعی‌ات عوض کن (بدون اسلش آخر). این آدرس توی sitemap، متادیتای سئو و لینک اشتراک‌گذاری تلگرام استفاده می‌شود.

## کار روزانه — اضافه کردن کتاب جدید

فایل `data/books.json` را باز کن. **همیشه آیتم جدید را بعد از علامت `[` (یعنی اول لیست) اضافه کن، نه آخر.** چون صفحه خانه «جدیدترین‌ها» را با شماره `id` بزرگ‌تر تشخیص می‌دهد — پس `id` کتاب جدید باید از همه بزرگ‌تر باشد.

```json
{
  "id": 10,
  "slug": "the-alchemist",
  "genre": "roman",
  "lang": "tr",
  "rate": 4.8,
  "isTop": true,
  "title": "کیمیاگر",
  "author": "پائولو کوئلیو",
  "cover": "✨",
  "coverBg": "linear-gradient(135deg,#11569A,#7EC8FF)",
  "tag": "رمان و داستان",
  "summary": "خلاصه کوتاه کتاب که در کارت‌های لیست نمایش داده می‌شود...",
  "review": "نقد کوتاه کتاب‌باز روی این کتاب...",
  "content": "<p>پاراگراف اول محتوای کامل‌تر، برای صفحه مجزای کتاب.</p><p>پاراگراف دوم...</p>",
  "pdf": "files/the-alchemist.pdf",
  "tg": "https://t.me/kmketab"
},
```

### نکته‌های فیلدها:
- **`slug`**: فقط حروف انگلیسی کوچک و خط فاصله، بدون فاصله. این بخشی از URL نهایی می‌شود (`/books/the-alchemist.html`). یکتا باشد.
- **`genre`**: یکی از `roman` | `science` | `personal` | `history` | `poetry`
- **`lang`**: `fa` (فارسی) یا `tr` (ترجمه)
- **`content`**: اختیاری! اگر وقت نداری محتوای کامل بنویسی، این فیلد را کلاً حذف کن — صفحه فقط با خلاصه و نقد ساخته می‌شود. اگر می‌خوای بنویسی، هر پاراگراف را با `<p>...</p>` بپوشان.
- **`pdf`**: مسیر فایل PDF که قبلش توی پوشه `files/` آپلود کردی.

بعد از اضافه کردن، **Commit changes** کن. Cloudflare Pages خودش `build.js` را اجرا می‌کند و:
- صفحه مجزای `/books/the-alchemist.html` ساخته می‌شود
- صفحه خانه و `/books.html` خودشان به‌روز می‌شوند
- `sitemap.xml` خودش این URL جدید را اضافه می‌کند
- کتاب‌های دیگر همان ژانر، خودکار به‌عنوان «مرتبط» در پایین صفحه این کتاب نشان داده می‌شوند

## اضافه کردن قسمت پادکست جدید

`data/podcast.json` را باز کن. آبجکت `featured` همیشه نشان‌دهنده **آخرین قسمت** است (همان که در صفحه پادکست بالا پخش می‌شود). آرایه `episodes` لیست کامل قسمت‌ها برای نمایش است.

برای قسمت جدید:
1. فایل mp3 را در پوشه `audio/` آپلود کن (مثلاً `audio/podcast-ep13.mp3`)
2. `featured` را با اطلاعات قسمت جدید جایگزین کن
3. یک آیتم جدید به **اول** آرایه `episodes` اضافه کن

```json
{
  "featured": {
    "title": "قسمت ۱۳ — عنوان قسمت جدید",
    "desc": "توضیح کوتاه این قسمت...",
    "audioFile": "audio/podcast-ep13.mp3",
    "durationSec": 2500
  },
  "episodes": [
    { "num": "۱۳", "title": "قسمت ۱۳ — عنوان قسمت جدید", "audioFile": "audio/podcast-ep13.mp3", "durationSec": 2500, "date": "این هفته" },
    { "num": "۱۲", "title": "...قسمت قبلی...", ... }
  ]
}
```

✅ صفحه خانه **هیچ پخش‌کننده‌ای ندارد** — فقط یک کارت معرفی با لینک به `/podcast.html` نشان می‌دهد. پخش‌کننده mp3 واقعی فقط در خود صفحه پادکست است (طبق درخواستت).

## اضافه کردن کتاب صوتی جدید

`data/audiobooks.json` را باز کن. دو حالت داری:

### حالت ۱ — فایل یک‌تکه (`"mode": "single"`)
```json
{
  "id": 9,
  "slug": "the-alchemist-audio",
  "genre": "roman",
  "title": "کیمیاگر",
  "author": "پائولو کوئلیو",
  "narrator": "با صدای رضا کریمی",
  "cover": "✨",
  "coverBg": "linear-gradient(135deg,#11569A,#7EC8FF)",
  "tag": "رمان و داستان",
  "rate": "۴.۸",
  "mode": "single",
  "audioFile": "audio/the-alchemist.mp3",
  "duration": "۴ ساعت ۲۰ دقیقه",
  "durationSec": 15600
}
```

### حالت ۲ — فصل‌بندی‌شده (`"mode": "chapters"`)
```json
{
  "id": 9,
  "slug": "the-alchemist-audio",
  "genre": "roman",
  "title": "کیمیاگر",
  "author": "پائولو کوئلیو",
  "narrator": "با صدای رضا کریمی",
  "cover": "✨",
  "coverBg": "linear-gradient(135deg,#11569A,#7EC8FF)",
  "tag": "رمان و داستان",
  "rate": "۴.۸",
  "mode": "chapters",
  "chapters": [
    { "title": "فصل ۱ — رویای سانتیاگو", "file": "audio/alchemist-ch1.mp3", "durationSec": 1800 },
    { "title": "فصل ۲ — سفر به مصر", "file": "audio/alchemist-ch2.mp3", "durationSec": 2100 }
  ],
  "duration": "۴ ساعت ۲۰ دقیقه", "durationSec": 15600
}
```

فایل‌های mp3 (یک‌تکه یا هر فصل جدا) را در پوشه `audio/` آپلود کن، اسم انگلیسی و بدون فاصله بگذار. بعد از Commit، صفحه `/audiobooks/the-alchemist-audio.html` خودش ساخته می‌شود — با پخش‌کننده دارای دکمه‌های ⏪ ۳۰ ثانیه عقب و ⏩ ۳۰ ثانیه جلو، و اگر فصل‌بندی باشد، فهرست فصل‌های قابل‌کلیک هم زیرش می‌آید.

## چرا این معماری به بازدید و ماندگاری کمک می‌کند؟

- **URL مجزا برای هر کتاب** → گوگل هر کتاب را جدا ایندکس می‌کند، یعنی جستجوهای دقیق (مثل «نقد کیمیاگر») می‌توانند مستقیم به همان صفحه برسند.
- **کتاب‌های مرتبط در پایین هر صفحه** → وقتی کاربر از تلگرام وارد یک کتاب می‌شود، بلافاصله ۳ کتاب هم‌ژانر دیگر را می‌بیند و به احتمال زیاد کلیک می‌کند → زمان ماندن در سایت بالا می‌رود.
- **دکمه اشتراک‌گذاری در تلگرام** روی هر صفحه کتاب → کاربران می‌توانند خودِ صفحه (نه فقط کانال) را برای دوستانشان بفرستند.
- **صفحه پادکست و کتاب صوتی مجزا با پخش‌کننده واقعی** → کاربر دلیلی برای ماندن روی سایت دارد، نه فقط خوانده و رفتن.
- **Sitemap همیشه به‌روز و خودکار** → گوگل سریع‌تر صفحات جدید را پیدا می‌کند.

## مرحله نهایی: ثبت در Google Search Console

۱. برو به [search.google.com/search-console](https://search.google.com/search-console)
۲. دامینت را اضافه کن و مالکیتش را تأیید کن (از طریق DNS که قبلاً روی Cloudflare داری، ساده است)
۳. توی بخش **Sitemaps**، آدرس `https://دامینت.com/sitemap.xml` را ثبت کن

از این به بعد، هر بار که کتاب جدید اضافه می‌کنی و push می‌کنی، گوگل خودش طی چند روز صفحه جدید را پیدا و ایندکس می‌کند.
